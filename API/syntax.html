Identifier¶

class Identifier(full_name: str, stream_range: Streams.StreamRange.StreamRange=None)[source]¶

Bases: Syntax.Code.Code

An identifier with a syntactic name, and an extra (called semantic) name.

Parameters:
  • full_name – The identifier’s full name, which should have the syntactic name and semantic name separated by a hash character. As in syntactic_name#semantic_name. If there is no semantic name, the hash can be omitted. If there is no semantic name but the syntactic name has a hash, just append a hash to it.
  • stream_range – The range where the identifier appears in.
full_name[source]¶

Returns the full name (syntactic name plus hash plus semantic name) if semantic name is non-empty, or else just the syntactic name.

Punctuator¶

class Punctuator(child_node, punctuation, skip_count, end_punctuation_marker=None)[source]¶

A node that should contain a single element, usually itself a node, whose punctuation is yet unprocessed.

Parameters:
  • child_node – The unprocessed child node.
  • punctuation – A list of elements of the child node that are punctuation which should be processed.
  • skip_count – The arrangement of punctuation of the child node should ignore these many elements; so, for instance, in lyc, the parsing of :literal:head(a b, c d) creates a form with children head a b , c d, and the , punctuation should group together a and b, but not head. So skip_count is 1. Also in lyc, the parsing of :literal:head[a b, c d] creates a form with the children @[] head a b , c d, and the , punctuation should not group together @[] or head. So skip_count is 2.
  • end_punctuation_marker – An element marking the position after the last element in the child_node to which

punctuation applies. Usually the INDENT token of an indented code block.

NodeIterator¶

class NodeIterator(first_element: Syntax.Node.Element)[source]¶

An iterator over the elements of a node. After iterating over some element, it is OK to change the node up to (and including) that element, but changing the node after that element might result in unexpected behavior (the iterator’s behavior becomes undefined).
